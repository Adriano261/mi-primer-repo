¿Qué es un patrón de diseño Web?

Son patrones que evitan la reiteración en la búsqueda de soluciones a problemas ya conocidos y solucionados anteriormente en entornos Web. Permiten estandarizar el modo en que se realiza el diseño tambien son patrones que facilitar el aprendizaje de las nuevas generaciones de diseñadores condensando conocimiento ya existente.

Tipos de patrones de diseño Web

  1. Cliente Web Delgado
    • Se utiliza cuando los clientes tienen poco poder de computación o no se tiene control sobre su configuración
    • Toda la lógica de trabajo se ejecuta en el servidor
    • El más utilizado en aplicaciones para Internet
  2. Cliente Web Grueso
    • Extiende el patrón Cliente Web Delgado mediante el uso de scripts u objetos del lado del cliente
    • El cliente ejecuta lógica de trabajo
    • Permite establecer interfaces de usuario sofisticadas
  3. Web Object (Distribuido)
    • Se basa en la utilización de la tecnología Web como parte de un sistema cliente/servidor de objetos distribuidos
    • Es apropiado para ser usado en ambientes donde hay control sobre la configuración del visualizador y la conexión de red (Intranet)

Soluciones

Un patrón de diseño es una solución probada y reutilizable a un problema común que se presenta en el desarrollo de software o diseño de interacción.

Curiosidades

Las curiosidades sobre los patrones de diseño incluyen su origen en la arquitectura (Christopher Alexander), que no son código listo para usar, sino plantillas de soluciones reutilizables. También son importantes porque crean un lenguaje común entre desarrolladores, permiten la reutilización de conocimiento y la adaptación a diferentes contextos, y existen tres categorías principales: patrones creacionales, estructurales y de comportamiento.

Caracteristicas

  • No son código literal:

    • Los patrones de diseño no son fragmentos de código que se copian y pegan, sino conceptos o descripciones de soluciones que deben adaptarse a las necesidades específicas de cada proyecto.

  • Lenguaje común:

    • Proporcionan un vocabulario compartido entre los diseñadores, lo que facilita la comunicación, el debate sobre alternativas y la estandarización del diseño.

  • Reutilización de conocimiento:

    • Permiten a los desarrolladores aprovechar la experiencia acumulada, evitando "reinventar la rueda" al solucionar problemas ya conocidos y demostrados como efectivos.
